# iHeater Link Pulse Protocol

## Назначение

Протокол `ESP32 Link -> STM32 iHeater` для минимальной передачи команды нагрева через один цифровой вход.

Идея:

- `claim` не меняем;
- портал пока не меняем;
- `Link` принимает состояние печати от `Bambu`;
- `Link` передаёт в `STM32` только команду `OFF` или целевую температуру;
- физический канал: один цифровой GPIO `ESP32 -> STM32 trigger input`.

## Физический уровень

- Линия в idle удерживается в `LOW`.
- Один кадр передаётся периодически, раз в `1 секунду`.
- Начало кадра определяется длинной паузой `SYNC LOW`.

Рекомендуемые тайминги MVP:

- `SYNC LOW`: `120 ms`
- `PULSE HIGH`: `2 ms`
- `PULSE LOW`: `2 ms`
- `FRAME PERIOD`: `1000 ms`

## Кодирование

После `SYNC LOW` передаётся пачка коротких импульсов.

Количество импульсов = код команды.

Принятые коды MVP:

- `10` = `OFF`
- `45` = `45C`
- `50` = `50C`
- `55` = `55C`
- `60` = `60C`
- `65` = `65C`
- `70` = `70C`
- `75` = `75C`
- `80` = `80C`

## Правила декодирования на STM32

- Если была пауза `>= 100 ms`, это старт нового кадра.
- После этого STM32 считает фронты.
- Когда после пачки импульсов снова наступает длинная пауза, кадр считается завершённым.
- Количество импульсов интерпретируется как команда.

Если код не распознан:

- команда игнорируется;
- сохраняется последнее валидное значение.

Если валидные кадры не приходят `>= 3 секунд`:

- перейти в `OFF`.

## Поведение Link

- `Bambu inactive / not connected / no heat request` -> код `10`
- `Bambu active` -> код ближайшей поддерживаемой температуры

Квантование температуры:

- использовать шаг `5C`
- диапазон `45..80C`
- всё ниже `45C` округлять к `45C`
- всё выше `80C` округлять к `80C`

## Примечания

- Это цифровой протокол без калибровки ADC.
- Его легко проверять логическим анализатором или осциллографом.
- Для MVP этого достаточно.
- Если позже потребуется больше параметров, можно расширить кадр, не меняя общую физику линии.
